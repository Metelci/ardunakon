SECTION 1: MAIN CONTROL SCREEN
====================================================================================================

OVERVIEW:
The Control Screen is the heart of Ardunakon, featuring:
- Dual joysticks for precise control
- Status cards showing connection info
- 4 customizable Aux buttons
- Real-time signal strength and battery indicators

----------------------------------------------------------------------------------------------------
JOYSTICK LAYOUT
----------------------------------------------------------------------------------------------------

LEFT JOYSTICK (Movement):
  - X-axis: Left/Right movement or steering
  - Y-axis: Forward/Backward (one motor in tank mode)
  - Range: 0-200 (100 is center/neutral)
  - Returns to center when released

RIGHT JOYSTICK (Throttle):
  - X-axis: Rotation or fine control
  - Y-axis: Throttle or second motor control
  - Range: 0-200 (100 is center/neutral)
  - Returns to center when released

JOYSTICK MODES:
  Car Mode:
    - Range: -100% to +100% (bidirectional)
    - Zero throttle at center
    - Perfect for RC cars with forward/reverse

  Drone/Boat Mode:
    - Range: 0% to 100% (unidirectional)
    - Zero throttle at bottom
    - Ideal for ESCs that don't support reverse

----------------------------------------------------------------------------------------------------
STATUS CARDS
----------------------------------------------------------------------------------------------------

Each device slot (Dev 1, Dev 2) has a status card showing:

CONNECTION STATUS (Color Coded):
  RED    - Disconnected
  YELLOW - Connecting...
  GREEN  - Connected and active

DEVICE INFO:
  Device Name: Bluetooth module name (e.g., "HC-06", "ArdunakonQ")
  MAC Address: Hardware address for identification
  Type: Classic Bluetooth or BLE

REAL-TIME TELEMETRY:
  Signal Strength: RSSI in dBm (-40 = Excellent, -90 = Poor)
  Battery Voltage: Read from Arduino (requires voltage divider on A0)
  Connection Time: How long connected
  Packet Stats: Sent/Received/Dropped

----------------------------------------------------------------------------------------------------

  1. Tap "Dev 1" button â†’ Select first device
  2. Wait for GREEN status
  3. Tap "Dev 2" button â†’ Select second device

SECTION 4: AUX BUTTON CONFIGURATION
====================================================================================================

AUX BUTTONS: 4 customizable buttons for special functions.

DEFAULT LAYOUT:
  AUX 1 (Top-Left)     - Button ID 1
  AUX 2 (Top-Right)    - Button ID 2
  AUX 3 (Bottom-Left)  - Button ID 3
  AUX 4 (Bottom-Right) - Button ID 4

----------------------------------------------------------------------------------------------------
CONFIGURING AUX BUTTONS
----------------------------------------------------------------------------------------------------

STEPS:
  1. Long-press any Aux button for 2 seconds
  2. Configuration dialog opens
  3. Set:
     - Button Label (e.g., "Lights", "Horn", "Camera")
     - Button Color (visual identification)
     - Command to send (Button ID 1-255)
     - Toggle vs Momentary mode
  4. Tap "Save"

BUTTON MODES:
  Momentary:
    - Sends PRESSED when tapped
    - Sends RELEASED when released
    - Good for: Horn, boost, temporary lights

  Toggle:
    - Sends PRESSED on first tap
    - Sends RELEASED on second tap
    - Good for: Headlights, modes, locks

----------------------------------------------------------------------------------------------------
AUX BUTTON PROTOCOL
----------------------------------------------------------------------------------------------------

When Aux button pressed, app sends:
  Packet: [0xAA, 0x01, 0x02, ButtonID, State, 0x00, 0x00, 0x00, CS, 0x55]

  ButtonID = 1-255 (configured in app)
  State = 1 (pressed) or 0 (released)

Arduino sketch receives button event:
  void handleButton(uint8_t buttonId, uint8_t pressed) {
    if (buttonId == 1 && pressed == 1) {
      digitalWrite(LED_PIN, HIGH); // Turn on LED
    }
  }

SECTION 5: JOYSTICK SETTINGS
====================================================================================================

SENSITIVITY ADJUSTMENT:
  Range: 10% - 200%
  Default: 100% (1:1 mapping)

USE CASES:
  10-50%  - Fine control (precision tasks, slow robots)
  100%    - Normal control (default)
  150-200%- High sensitivity (racing, fast response)

----------------------------------------------------------------------------------------------------
HOW TO ADJUST SENSITIVITY
----------------------------------------------------------------------------------------------------

GLOBAL SENSITIVITY:
  1. Tap "Settings" â†’ "Joystick Sensitivity"
  2. Adjust slider
  3. Test in real-time (joystick preview updates)
  4. Tap "Apply"

PER-AXIS SENSITIVITY:
  1. Tap "Advanced" in Settings
  2. Adjust Left X, Left Y, Right X, Right Y independently
  3. Example:
     - Left X: 50% (slow steering)
     - Right Y: 150% (fast throttle response)

DEADZONE:
  Small zone around center where joystick sends neutral value
  Default: 5%
  Adjustable: 0-20%

  Use Cases:
    - 0%: Maximum precision (racing drones)
    - 5%: Default (prevents drift)
    - 10-20%: Old/worn joystick calibration

----------------------------------------------------------------------------------------------------
COMMON AT COMMANDS
----------------------------------------------------------------------------------------------------

HC-05 / HC-06:
  AT             â†’ Test connection (responds "OK")
  AT+VERSION     â†’ Get firmware version
  AT+NAME?       â†’ Get device name
  AT+NAMEMyBot   â†’ Set device name to "MyBot"
  AT+BAUD4       â†’ Set baud rate to 9600
  AT+PIN1234     â†’ Set pairing PIN to 1234
  AT+RESET       â†’ Restart module

HM-10 / HC-08:
  AT             â†’ Test connection
  AT+VERS?       â†’ Get firmware version
  AT+ADDR?       â†’ Get MAC address
  AT+NAMEDrone1  â†’ Set name to "Drone1"
  AT+BAUD0       â†’ Set baud to 9600
  AT+RESET       â†’ Restart

IMPORTANT:
  - HC-05/HC-06 require CR+LF line ending (\r\n)
  - HM-10/HC-08 require NO line ending
  - Set correct line ending in Terminal settings!

SECTION 8: STATUS INDICATORS
====================================================================================================

LED INDICATORS (on Bluetooth modules):

HC-05 / HC-06:
  Blinking (2x per sec)  - Not connected, advertising
  Solid ON               - Connected to phone
  Blinking (1x per 2sec) - In AT command mode (HC-05 only)

HM-10 / HC-08:
  Blinking fast          - Not connected, advertising
  Blinking slow          - Connected
  Solid ON               - Data transfer active

Arduino Built-in LED (in sketches):
  Blinking (1Hz)         - Waiting for connection
  Solid ON               - Connected and receiving data
  Rapid blinking         - Error / timeout

----------------------------------------------------------------------------------------------------
APP STATUS COLORS
----------------------------------------------------------------------------------------------------

CONNECTION STATUS:
  RED    - Disconnected or error
  YELLOW - Connecting (trying all methods)
  GREEN  - Connected and active

SIGNAL STRENGTH:
  Green bars  - Strong (-40 to -60 dBm)
  Yellow bars - Medium (-60 to -75 dBm)
  Red bars    - Weak (-75 to -90 dBm)
  Gray        - No signal / disconnected

BATTERY INDICATOR:
  Green  - > 11.5V (healthy)
  Yellow - 10.5 - 11.5V (low)
  Red    - < 10.5V (critical, recharge!)

====================================================================================================
TIPS & TRICKS
====================================================================================================

TIP 1: SAVE BATTERY
  - Lower packet rate to 10Hz
  - Disable "Keep Screen On"
  - Use Drone mode (less joystick updates)

TIP 2: IMPROVE RANGE
  - Keep phone and module in line-of-sight
  - Avoid 2.4GHz WiFi interference
  - Use external antenna on HC-05 if available

TIP 3: FASTER CONNECTION
  - Save devices to profiles
  - Disable "Try all UUID variants" in Settings
  - Unpair old devices from Android Bluetooth

TIP 4: EMERGENCY STOP
  - Shake phone to trigger E-STOP (if enabled in settings)
  - Or tap both joysticks simultaneously

TIP 5: MULTIPLE PROJECTS
  - Create separate profile for each RC project
  - Name profiles clearly ("Red Car", "Drone #2")
  - Use different Aux button colors per profile
